In a nutshell

Fact-checking models often cheat: they reach high accuracy by reading surface patterns rather than reasoning about evidence. A sensible fix is to weight evidence by how trustworthy its source is. But to claim a model actually uses source trust, you need a test that only a trust-using model can pass. We show that the obvious way to build such a test — pairing the same claim with a high-trust and a low-trust source — secretly leaks the answer: the low-trust evidence was written in a different style, so a model can read the style instead of the source. After fixing the leak, the result flips: a model that ignores source trust drops to near-chance (macro-F1 0.385), while a trust-aware model wins (0.918). Mid-tier boundary probes further reveal that access to the trust scalar alone is not enough — trust must be reasoned over continuously, not merely thresholded.

The original evaluation does not measure source trust

Three reported successes each collapse under inspection.

Synthetic split

Low-trust evidence uses a rumour register (“an anonymous post claimed…”), high-trust uses an official one. So text style = verdict. A trust-blind model scores 0.996; the template field predicts the label at 0.99.

AVeriTeC accuracy

The set is ~66% “refuted”. Reported accuracy 0.665 is the majority-class prior; its macro-F1 is 0.265. A real classifier collapses to the majority class.

Simulation split

Constant trust and weight, balanced labels, closed-world numeric match → accuracy 1.000. The epistemic formula contributes nothing; it only inflates the average.

The Epistemic Confidence formula

EC = 1 − (1 − ST)EW × IS

ST — source trust ∈ [0,1]: registry-assigned reliability of the source
EW — evidence weight: perception 0.95, testimony 0.80, inference 0.55
IS — inference strength ∈ [0,1]: logical binding of evidence to claim

Benchmark label rule: EC ≥ 0.75 → SUPPORTED or REFUTED (depending on stance); else NOT_ENOUGH_EVIDENCE. High-trust sources (ST ≥ 0.85) clear the threshold; low-trust (ST ≤ 0.45) fall well below it; mid-tier probes (ST ∈ {0.50, 0.62, 0.72}) sit in the gap — all below 0.75, all labelled NEE. (The trained model uses a HybridVerdictHead when EC < 0.75 — see model description below.)

A trust-isolating benchmark

We rebuild the test so source trust is the only feature that can determine the label.

Design (2,200 records)
  • Evidence text byte-identical across trust tiers, neutral register.
  • Source category held constant (category carries no signal).
  • Trust values held out at test: model must reason over a continuous scalar.
  • Mid-tier probes (ST ∈ {0.50, 0.62, 0.72}) test whether the model can generalise beyond binary thresholding.
Verification

1028 of 1068 distinct evidence texts appear with more than one gold label (the 40 exceptions are mid-tier boundary probes, NEE-only by design). Source category, stance, and evidence text all predict the verdict at chance (0.50). Source trust is the only predictive signal.

The inversion

Source trust helps only once the split stops leaking. The same signal flips from winning to losing.

Honest limitations

The corrected benchmark is a controlled instrument: single-evidence, single evidence-type, fictional-claim, single-category by design, so it measures trust sensitivity, not open-web realism. GNN results report mean ± std across three independent training runs; v2-HGNN and v3-NLI show similar variance (σ = 0.017 and σ = 0.019 respectively), with v3-NLI's slight instability attributable to competing NLI signals on template text. A decision-path breakdown confirms the EC symbolic path is identical and perfect for both models (40% of test decisions, 100% accuracy); the gap localises entirely to the HybridVerdictHead fallback, where v3-NLI achieves 0.852 accuracy vs 0.870 for v2-HGNN — all errors are supported or refuted predictions on records that should be NEE. The mid-tier boundary zone (macro-F1 0.204) reveals a gap in continuous trust reasoning that warrants a purpose-built model. Both EW (testimony, 0.80) and IS (inference strength, 1.0 — all evidence directly states the claim) are held constant; neither axis is tested here. An evidence-type-isolating benchmark (varying EW) and an inference-strength-isolating benchmark (varying IS through indirect or partial evidence) are the natural next steps. The IS axis also suggests a rehabilitation path for v3-NLI: NLI probabilities are appropriate IS estimators and may improve over v2-HGNN in a benchmark where IS genuinely varies.
