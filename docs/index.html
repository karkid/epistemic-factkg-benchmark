In a nutshell

Fact-checking models often cheat: they reach high accuracy by reading surface patterns rather than reasoning about evidence. A sensible fix is to weight evidence by how trustworthy its source is. But to claim a model actually uses source trust, you need a test that only a trust-using model can pass. We show that the obvious way to build such a test — pairing the same claim with a high-trust and a low-trust source — secretly leaks the answer: the low-trust evidence was written in a different style, so a model can read the style instead of the source. After fixing the leak, the result flips: a model that ignores source trust drops to chance, while a trust-aware model wins. The lesson is about measurement, not just models — and it took two attempts to get the benchmark right.

The original evaluation does not measure source trust

Three reported successes each collapse under inspection.

Synthetic split

Low-trust evidence uses a rumour register (“an anonymous post claimed…”), high-trust uses an official one. So text style = verdict. A trust-blind model scores 0.996; the template field predicts the label at 0.99.

AVeriTeC accuracy

The set is ~66% “refuted”. Reported accuracy 0.665 is the majority-class prior; its macro-F1 is 0.265. A real classifier collapses to the majority class (predicts NEI 1 / 462 times).

Simulation split

Constant trust and weight, balanced labels, closed-world numeric match → accuracy 1.000. The epistemic formula contributes nothing; it only inflates the average.

A trust-isolating benchmark

We rebuild the test so source trust is the only feature that can determine the label.

Design
  • Evidence text byte-identical across trust tiers, neutral register.
  • Source category held constant (so the category feature carries no signal).
  • Trust values held out at test, so the model must use trust as a continuous signal.
The lesson (it took two tries)

Text isolation alone was not enough: a first version still let a trust-blind model reach 0.82 macro-F1 by using source category as a trust proxy. Isolating a feature means controlling its proxies too. Verification: 747/747 texts appear with more than one label; every non-trust feature predicts at chance.

The inversion

Source trust helps only once the split stops leaking. The same signal flips from winning to losing.

Honest limitations

The corrected benchmark is a controlled instrument: single-evidence, single evidence-type, fictional-claim, single-category by design, so it measures trust sensitivity, not open-web realism. The trust-aware probe score of 1.000 reflects a label that is a deterministic function of trust on this split; the content is the contrast with the trust-blind 0.447, not the absolute number. Results are a single run (multiple seeds belong in a camera-ready). The orthogonal evidence-type axis is held fixed here and remains untested — an analogous evidence-type-isolating benchmark is the natural next step.
